# 技術調査・検討リスト

Synthify のアーキテクチャや機能実装において、今後詳細な調査や深掘りが必要な事項を管理します。

## 1. ストレージ・アップロード

### 1.1 Resumable Upload (再開可能アップロード) の最適化
- **現状:** バックエンドでセッションを開始し、フロントエンドから `PUT` で一括送信する方式を採用。
- **調査事項:**
    - GB 級の超巨大ファイルにおける、フロントエンド側でのチャンク（分割）送信の実装要件。
    - ネットワーク断絶時の再開ロジック（`Content-Range` ヘッダーの制御）。
    - 署名付き URL (Signed URL) と Resumable Upload を組み合わせた際の本番環境（GCS）での挙動確認。

### 1.2 ストレージ互換性 (S3 vs GCS)
- **現状:** GCS (およびそのエミュレータ) の JSON API に準拠。
- **調査事項:**
    - MinIO などの S3 互換ストレージに移行する場合の、アップロードプロトコルの差異の吸収。
    - マルチクラウド対応を見据えた抽象化レイヤーの設計。

## 2. ジョブ処理・スケーラビリティ

### 2.1 大容量ドキュメントのパース
- **現状:** Worker がファイルを一括で読み込み処理。
- **調査事項:**
    - メモリ制限を超えるファイルのストリーム処理。
    - ページ数の多い PDF 等の分割処理と並列処理のオーケストレーション。

## 3. セキュリティ・認証

### 3.1 アップロードセッションの保護
- **調査事項:**
    - 発行した Resumable URL の有効期限と、不正利用防止のための署名検証の強化。
    - ユーザーごとのストレージクォータ（容量制限）の適用タイミング。
