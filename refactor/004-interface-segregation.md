# Refactor 004: インターフェース分離（疎結合の徹底）

## 現状
`worker.Repository` が非常に多くのリポジトリを継承した「巨大なインターフェース」になっています。

## 課題
- 各関数が実際に使用しないメソッドまで要求してしまっている。
- モック作成やテストの際、不要なメソッドまで考慮する必要があり、テストコードが複雑になる。

## 解決策
- インターフェース分離の原則（ISP）に基づき、各コンポーネントが必要な最小限のインターフェースのみを受け取るようにリファクタリング。
- `shared/repository` で定義された個別のインターフェースを直接コンポーネントに注入。
